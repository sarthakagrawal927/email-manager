# Foundry Evidence

Privacy-safe build, sync, and auth evidence for the Foundry (`saas-maker`).
Generated by `scripts/foundry-evidence.mjs` and uploaded as a CI artifact on
every push to `main` and weekly (Mondays 06:00 UTC).

## Why

Email Manager processes private Gmail data. The Foundry needs to know whether
the project builds, whether auth is configured, and whether the sync lifecycle
is healthy — without receiving any email content, addresses, subjects,
credentials, or tokens. This script produces aggregate, sanitized evidence only.

## Run locally

```bash
pnpm foundry:evidence
# → writes foundry-evidence.json (gitignored)
```

The script runs `typecheck`, `lint`, `test:unit`, and `build`, then verifies
sync lifecycle invariants and sanitized error handling by static analysis of
the source. No dev server, no OAuth, no network calls to Gmail.

## Privacy guarantees

The evidence script NEVER reads:

- IndexedDB email records (bodies, subjects, from, to, snippets, embeddings)
- OAuth tokens, session cookies, or API keys
- Gmail API responses
- User identifiers or email addresses

It only reads source files (`src/lib/*.ts`, `src/worker.ts`, `wrangler.toml`)
and checks for the presence of variable names (not values).

## Sync lifecycle invariants

The script verifies these invariants by static analysis:

1. **Bounded work** — `SYNC_MAX_PAGE = 500` and `Math.min(SYNC_MAX_PAGE, ...)`
2. **Cursor/watermark** — `InboxSyncMeta.nextPageToken` persisted between runs
3. **Concurrency** — `syncLockRef` in `MailboxStoreProvider` prevents concurrent syncs
4. **Idempotency** — `storeEmails` uses `tx.store.put(e)` keyed by `id` (not `add`)
5. **Timeout** — `AbortSignal` honored in fetch calls
6. **Retry maximum** — `gmailFetch` retries 3× with exponential backoff
7. **Freshness** — `INBOX_STALE_MS` (1 hour) + `isInboxStale()`
8. **Durable failure** — `InboxSyncMeta.lastError` records sanitized stage + class + timestamp; cleared on success

## Foundry ingestion

The Foundry (`saas-maker`) can fetch the `foundry-evidence` workflow artifact to
track build/live/job status across the fleet. No hosted data plane is created —
the artifact is a static JSON file with 30-day retention. See
[`foundry.projects.json`](https://github.com/sarthakagrawal927/saas-maker/blob/main/foundry.projects.json)
for the fleet registry.
